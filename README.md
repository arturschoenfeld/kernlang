# Kern Programming Language
Kern (or Kernlang) is a systems programming language frontend that is inspired by C, Zig and Rust. Kern serves as a personal exploration and research of demand-driven compilation. It also aims to provide an easily readable syntax that reduces cognitive load and lots of punctuation.

## Motivation
Kern's purpose for me is not only to learn about compilers in general, but also to understand advances techniques, such as incremental compilation or demand-driven query-based systems. This way, I want to avoid as much recompilation as possible when only small changes have been made.

As for the syntax, I try to develop something that is more readable, less punctuated and less mentally taxing. Not only is it important to reduce the amount of intrinsic language features, but also to shorten the amount of punctuators to get something done. It should be easier to convey the gist (or in German, the **Kern**) of a code sequence, as long as the user is interested in the brevity of a code without resoring to cryptic punctuation sequences that may look intimidating to beginners and mentally taxing to overworked and tired intermediates and professionals. This is a difficult task on its own, as people have different preferences.
